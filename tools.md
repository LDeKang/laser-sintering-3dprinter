# TOOLS.md - 工具记忆 / 踩坑记录

## 工具使用经验

### Git / GitHub

**最佳实践**:
- 每次修改后及时commit
- 写有意义的commit message
- 重要节点打tag

**踩坑记录**:
- ❌ 不要等所有内容写完再push，容易丢失
- ✅ 小步快跑，频繁提交

### Feishu Doc (飞书文档)

**最佳实践**:
- 先创建文档，再追加内容
- 大内容分批次append，避免超时
- 表格内容先规划好再写入

**踩坑记录**:
- ❌ 不要试图一次性写入超大内容（会失败）
- ❌ 不要依赖文档覆盖功能（不支持）
- ✅ 分批append，每次检查是否成功
- ✅ 重要文档创建"完整版"副本

### Markdown

**最佳实践**:
- 使用标准Markdown语法
- 表格对齐，便于阅读
- 代码块标注语言

**踩坑记录**:
- ❌ ASCII图表在某些平台显示错乱
- ✅ 使用简单表格替代复杂图表
